# 结算与战绩增强设计

## 目标

让胜利与死亡结算界面更像正式可上线游戏的复盘界面，让玩家能快速理解“这一局表现如何、有没有新纪录、下一步该做什么”。

## 范围

本轮只增强结算展示和轻量统计，不改变战斗、地图、卡牌、敌人、遗物、药水数值。

主要修改：

- `js/ui/chapter-ui.js`
- `css/main.css`
- 如有必要，补充 `test/regression-test.js`

## 胜利结算

胜利界面显示：

- 总分
- 评分等级：S / A / B / C
- 是否刷新最高分
- 连胜数
- 本局摘要：职业、模式、难度、章节/层数、击杀、Boss、剩余 HP、遗物数、牌组数
- 得分拆解：层数、击杀、Boss、生命、遗物、精简牌组、模式倍率、难度倍率
- 明确按钮：返回主菜单

最高分应在胜利界面显示前保存，这样玩家不必点击按钮才记录成绩。

## 死亡结算

死亡界面显示：

- 死亡标题与简短评价
- 死亡位置：第几章、第几层
- 本局摘要：职业、模式、难度、击杀、Boss、遗物、牌组数
- 历史死亡次数
- 明确按钮：
  - 重新挑战当前节点
  - 结束本局并返回主菜单

死亡应清空连胜，保留当前“重新挑战当前节点”的逻辑。

## UI 风格

沿用现有暗黑地牢风格，不引入新库。

设计原则：

- 弹窗最大宽度比现在略大，容纳复盘信息。
- 使用网格信息块展示统计项，避免一长串文字。
- 胜利用金色强调，新纪录和评分有明显视觉层级。
- 死亡用危险色强调，但按钮仍保持清晰主次。
- 小屏幕允许弹窗内容滚动，不溢出视口。

## 非目标

本轮不做：

- 成就系统
- 历史对局列表
- 图鉴
- 新手引导
- 新卡牌、敌人、事件或 Boss
- 上传 GitHub 或自动 commit

## 验证

完成后运行：

- `node test/batch2-test.js`
- `node test/batch3-test.js`
- `node test/batch4-test.js`
- `node test/regression-test.js`
- 所有 `.js` 文件语法检查
- `git diff --check`

浏览器验证：

- 打开 `index.html`
- 检查主菜单无控制台错误
- 直接触发胜利结算，确认评分、新纪录、得分拆解、返回主菜单正常
- 直接触发死亡结算，确认重试/结束按钮正常
- 检查 1365x768 和 390x844 视口下结算弹窗没有明显溢出或遮挡
